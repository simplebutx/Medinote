## 프로젝트 소개

MediNote는 처방전 OCR, 약 정보 RAG 챗봇, 복약 일정 관리, 복약 기록 추적을 제공하는 AI 기반 복약 관리 서비스입니다.  
사용자는 처방전 이미지를 업로드하면 AI가 복약 정보를 추출하고, 백엔드는 이를 복약 일정으로 연결하여 개인화된 복약 관리 경험을 제공합니다.


## 핵심 설계 포인트

이 프로젝트에서는 AI 기능을 단순 부가 기능으로 붙이는 데 그치지 않고, OCR 결과를 복약 일정 도메인과 연결하고, 사용자 복약 데이터를 다시 챗봇 컨텍스트로 활용하는 순환 구조를 설계했습니다.  
이를 통해 처방전 인식, 복약 일정 생성, 복약 기록, 약 정보 질문 응답이 하나의 사용자 흐름으로 이어지도록 구현했습니다.



## 담당 역할

본 프로젝트에서 AI 기능 연동과 복약 도메인 백엔드 구현을 중심으로 담당했습니다.  
처방전 이미지 OCR, 약 정보 기반 RAG 챗봇, 복약 일정 자동 생성, 복약 기록 조회 흐름을 연결하여 사용자가 처방전 업로드부터 복약 관리까지 하나의 서비스 안에서 사용할 수 있도록 구현했습니다.

## 주요 기여

### AI 서버 및 OCR 파이프라인

- FastAPI 기반 AI 서버를 구성하고, 백엔드 서비스와 연동되는 AI API를 설계했습니다.
- 처방전 이미지를 OCR로 분석하여 약 이름, 복용량, 복용 횟수, 복용 기간 등의 복약 정보를 추출하는 파이프라인을 구현했습니다.
- 이미지 품질 평가, 회전 보정, OCR confidence 요약 로직을 추가하여 다양한 처방전 이미지에서도 안정적으로 정보를 추출할 수 있도록 개선했습니다.
- OCR 결과를 구조화된 JSON 형태로 변환하여 백엔드에서 복약 일정 생성에 활용할 수 있도록 설계했습니다.

### 약 정보 RAG 챗봇

- Qdrant 벡터 DB와 임베딩 모델을 활용하여 약 정보 문서 검색 기반 RAG 챗봇을 구현했습니다.
- 사용자가 특정 약을 멘션하거나 복약 일정 관련 질문을 입력하면, 약 정보와 사용자 복약 컨텍스트를 기반으로 답변하도록 구성했습니다.
- 단순 LLM 응답이 아니라 검색된 문서 근거를 기반으로 답변하도록 설계하여 의료 정보 답변의 신뢰성을 높였습니다.
- 답변 유형을 일반 답변, fallback, 약사 상담 권장으로 구분하여 안전한 응답 흐름을 만들었습니다.

### 복약 관리 백엔드

- Spring Boot 기반 복약 서비스에서 처방전 OCR 결과를 복약 일정 도메인과 연결하는 API 흐름을 구현했습니다.
- 복약 일정, 복약 시간, 복약 기록을 관리하는 도메인 모델과 CRUD API를 설계했습니다.
- 사용자의 복약 일정 데이터를 챗봇 컨텍스트로 제공하여 AI가 개인화된 복약 관련 답변을 생성할 수 있도록 연동했습니다.
- 약 정보 검색, 복약 주의 태그, 처방 분석 기능을 AI 서버와 백엔드 서비스 간 API 호출 구조로 연결했습니다.



## 문제 해결 경험

- OCR 결과가 항상 정확하지 않을 수 있기 때문에 이미지 품질, 회전 방향, 토큰 confidence를 함께 기록하여 분석 가능한 구조로 개선했습니다.
- LLM이 의료 정보를 임의로 생성하지 않도록 Qdrant 기반 문서 검색 결과를 컨텍스트로 제공하고, 답변 유형을 구분하여 약사 상담이 필요한 경우를 분리했습니다.
- AI 서버, 복약 백엔드, 상담 백엔드가 분리된 구조에서 서비스 간 API 호출과 Docker 네트워크 구성을 정리하여 로컬 및 배포 환경에서 동일한 흐름으로 동작하도록 구성했습니다.
