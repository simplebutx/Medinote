## 프로젝트 소개

**MediNote**는 처방전 OCR, 약 정보 RAG 챗봇, 복약 일정 관리, 복약 기록 추적 기능을 제공하는 AI 기반 복약 관리 서비스입니다.

사용자가 처방전 이미지를 업로드하면 AI 서버가 복약 정보를 추출하고, 백엔드는 해당 결과를 복약 일정 도메인과 연결하여 개인화된 복약 관리 흐름을 제공합니다.

---

## 핵심 설계 포인트

* OCR 결과를 단순 텍스트 추출에 그치지 않고 복약 일정 생성 흐름과 연결
* 사용자 복약 일정 및 복약 기록 데이터를 챗봇 컨텍스트로 활용
* 처방전 인식 → 복약 일정 생성 → 복약 기록 → 약 정보 질문 응답으로 이어지는 순환 구조 설계
* AI 기능을 서비스 외부 부가 기능이 아닌 복약 관리 도메인의 핵심 흐름으로 통합

---

## 담당 역할

AI 기능 연동과 복약 도메인 백엔드 구현을 중심으로 담당했습니다.

* FastAPI 기반 AI 서버 구성
* 처방전 OCR 파이프라인 구현
* Qdrant 기반 약 정보 RAG 챗봇 구현
* OCR 결과 기반 복약 일정 자동 생성 흐름 연동
* 복약 일정, 복약 시간, 복약 기록 관리 API 설계 및 구현
* 사용자 복약 데이터를 챗봇 컨텍스트로 제공하는 백엔드 연동 구조 구현

---

## 주요 기여

### AI 서버 및 OCR 파이프라인

* FastAPI 기반 AI 서버 구성 및 백엔드 연동 API 설계
* 처방전 이미지에서 약 이름, 복용량, 복용 횟수, 복용 기간 등 복약 정보 추출
* 이미지 품질 평가, 회전 보정, OCR confidence 요약 로직 추가
* OCR 결과를 구조화된 JSON 형태로 변환하여 복약 일정 생성에 활용
* 다양한 처방전 이미지 입력에서도 분석 가능한 OCR 처리 흐름 구성

### 약 정보 RAG 챗봇

* Qdrant 벡터 DB와 임베딩 모델을 활용한 약 정보 검색 기반 RAG 챗봇 구현
* 사용자가 특정 약을 언급하거나 복약 일정 관련 질문을 입력하면 약 정보와 사용자 복약 컨텍스트를 함께 활용하도록 구성
* 단순 LLM 응답이 아닌 검색된 문서 기반 답변 구조 설계
* 답변 유형을 일반 답변, fallback, 약사 상담 권장으로 구분하여 안전한 응답 흐름 구성
* 의료 정보 답변에서 근거 부족 또는 위험 가능성이 있는 경우 상담 권장 흐름으로 분리

### 복약 관리 백엔드

* Spring Boot 기반 복약 서비스에서 OCR 결과를 복약 일정 도메인과 연결
* 복약 일정, 복약 시간, 복약 기록을 관리하는 도메인 모델 및 CRUD API 설계
* 사용자 복약 일정 데이터를 챗봇 컨텍스트로 제공하는 연동 구조 구현
* 약 정보 검색, 복약 주의 태그, 처방 분석 기능을 AI 서버와 백엔드 서비스 간 API 호출 구조로 연결

## 문제 해결 경험

### OCR 결과 신뢰도 문제

처방전 이미지의 품질, 회전 방향, OCR confidence에 따라 결과 정확도가 달라지는 문제가 있었습니다.

이를 해결하기 위해 이미지 품질 평가, 회전 보정, 토큰별 confidence 요약 정보를 함께 기록하도록 개선했고, OCR 결과를 백엔드에서 검토 및 활용하기 쉬운 구조화된 JSON 형태로 변환했습니다.

### 의료 정보 답변의 안정성 문제

LLM이 의료 정보를 임의로 생성할 가능성을 줄이기 위해 Qdrant 기반 문서 검색 결과를 답변 컨텍스트로 제공했습니다.

또한 답변 유형을 일반 답변, fallback, 약사 상담 권장으로 구분하여 근거가 부족하거나 위험 가능성이 있는 질문은 상담 권장 흐름으로 분리했습니다.

### 서비스 간 연동 구조 정리

AI 서버, 복약 백엔드, 상담 백엔드가 분리된 구조에서 API 호출 흐름과 Docker 네트워크 구성을 정리했습니다.

이를 통해 로컬 개발 환경과 배포 환경에서 동일한 방식으로 서비스 간 통신이 이루어지도록 구성했습니다.
