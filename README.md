## 프로젝트 소개

**MediNote**는 처방전 OCR, 약 정보 RAG 챗봇, 복약 일정 관리, 복약 기록 추적, 약국 검색, 스마트 약통 알림 기능을 통합한 **AI 기반 복약 관리 서비스**입니다.

사용자가 처방전 또는 약봉투 이미지를 업로드하면 AI 서버가 약 이름, 복용량, 복용 횟수, 복용 기간 등의 복약 정보를 추출하고, 백엔드는 해당 결과를 복약 일정 도메인과 연결하여 개인화된 복약 관리 흐름을 제공합니다.

- 개발 기간: 2026.04 ~ 2026.06
- 참여 인원: 3명
- 담당 역할: 백엔드 / AI

## 핵심 설계 포인트

- OCR 결과를 단순 텍스트 추출에 그치지 않고 복약 일정 자동 생성 흐름과 연결
- 사용자 복약 일정 및 복약 기록 데이터를 챗봇 컨텍스트로 활용
- 처방전 인식 → 복약 일정 생성 → 복약 기록 → 약 정보 질문 응답으로 이어지는 순환 구조 설계
- AI 기능을 서비스 외부 부가 기능이 아닌 복약 관리 도메인의 핵심 흐름으로 통합
- 약 정보 답변에서 근거 부족 또는 위험 가능성이 있는 경우 약사 상담 권장 흐름으로 분리

## 주요 기여

### AI 서버 및 OCR 파이프라인

- FastAPI 기반 AI 서버를 구성하고 백엔드 서비스와 연동되는 AI API를 설계했습니다.
- Google Cloud Vision OCR을 활용해 처방전 및 약봉투 이미지에서 약 이름, 복용량, 복용 횟수, 복용 기간 등의 복약 정보를 추출했습니다.
- OCR 인식률을 높이기 위해 이미지 품질 평가, 회전 보정, 전처리 흐름을 구성했습니다.
- OCR 결과에서 텍스트와 좌표 정보를 추출한 뒤, 좌표 기반으로 줄 단위 텍스트를 재구성했습니다.
- OCR confidence 요약 정보를 함께 기록하여 결과 분석과 디버깅이 가능하도록 했습니다.
- 추출된 약품명을 데이터베이스의 의약품 정보와 매칭하여 복약 일정 자동완성에 활용할 수 있도록 구성했습니다.
- OCR 결과를 구조화된 JSON 형태로 변환하여 백엔드의 복약 일정 생성 흐름과 연결했습니다.
- 사용자가 자동완성된 복약 정보를 최종 확인 및 수정한 뒤 일정으로 저장할 수 있도록 연동했습니다.

### 약 정보 RAG 챗봇

- Qdrant 벡터 DB와 임베딩 모델을 활용해 약 정보 검색 기반 RAG 챗봇을 구현했습니다.
- 사용자가 특정 약을 언급하거나 복약 일정 관련 질문을 입력하면, 약 정보와 사용자 복약 컨텍스트를 함께 활용하도록 구성했습니다.
- 질문 의도를 분류한 뒤, 약 관련 문서 청크를 검색하고 관련성이 높은 문서를 LLM에 전달하는 흐름을 구현했습니다.
- 단순 LLM 응답이 아니라 검색된 근거 문서를 기반으로 답변하도록 설계했습니다.
- 답변 유형을 일반 답변, fallback, 약사 상담 권장으로 구분하여 의료 정보 답변의 안전성을 높였습니다.
- 근거 문서가 부족하거나 위험 가능성이 있는 질문은 단정적으로 답변하지 않고 상담 권장 흐름으로 분리했습니다.

### 복약 관리 백엔드

- Spring Boot 기반 복약 서비스에서 OCR 결과를 복약 일정 도메인과 연결했습니다.
- 복약 일정, 복약 시간, 복약 기록을 관리하는 도메인 모델과 CRUD API를 설계 및 구현했습니다.
- 사용자 복약 일정 데이터를 챗봇 컨텍스트로 제공하여 AI가 개인화된 복약 관련 답변을 생성할 수 있도록 연동했습니다.
- 약 정보 검색, 복약 주의 태그, 처방 분석 기능을 AI 서버와 백엔드 서비스 간 API 호출 구조로 연결했습니다.
- 공공데이터 API를 활용해 의약품 정보와 약국 정보를 관리하는 기능을 구현했습니다.
- 복약 일정과 복약 기록을 기반으로 알림 기능을 구현했습니다.

### IoT 스마트 약통 연동

- ESP32-S3 기반 스마트 약통과 백엔드 복약 관리 기능을 연동했습니다.
- Arduino IDE 기반으로 스마트 약통 펌웨어를 개발했습니다.
- 복약 일정에 맞춰 스마트 약통 알림이 동작하고, 복약 상태를 서비스 흐름과 연결할 수 있도록 구성했습니다.

## 문제 해결 경험

### 다양한 약봉투 양식에 대응하는 OCR 후처리

약봉투마다 표 구조와 헤더 표현이 달라 하나의 고정된 파싱 규칙으로는 안정적인 정보 추출이 어려웠습니다.  
이를 해결하기 위해 여러 추출 로직을 동시에 실행하고, 각 결과를 점수화하여 가장 신뢰도 높은 결과를 최종 선택하는 방식으로 후처리 구조를 개선했습니다.  
또한 새로운 양식이 추가되더라도 전체 로직을 수정하지 않고 추출 패턴만 확장할 수 있도록 코드를 분리했습니다.

### OCR 모델 비교를 통한 기술 선택

OCR 품질이 복약 일정 자동완성의 정확도에 직접적인 영향을 주기 때문에 Google Vision, Naver OCR, Azure OCR을 비교했습니다.  
핵심 키워드 추출 여부, 노이즈 수, 응답 시간, 글자 오류율을 기준으로 평가했고, Azure가 비용과 노이즈 측면에서 장점이 있음을 확인했습니다.  
다만 기존 후처리 로직과의 호환성 및 교체 비용을 고려해 현재는 Google Vision을 유지하고, 추후 운영 규모가 커질 경우 Azure 전환을 검토하는 방향으로 결정했습니다.

### RAG 챗봇 검색 품질 개선

약 관련 질문은 사용자 표현과 의약품 문서 표현이 다를 수 있어 단순 키워드 검색만으로는 정확한 근거 문서를 찾기 어려웠습니다.  
이를 해결하기 위해 질문에서 핵심 키워드와 유사 표현을 함께 추출하고, 키워드 매칭 점수와 벡터 유사도 점수를 함께 사용하여 문서 청크의 관련도를 계산했습니다.  
이를 통해 "술", "알코올", "음주"처럼 표현이 달라도 같은 의도에 해당하는 문서를 검색할 수 있도록 개선했습니다.

### 임베딩 및 LLM 모델 비교

RAG 검색 품질을 높이기 위해 OpenAI small, Solar Embedding, OpenAI large 임베딩 모델을 비교했습니다.  
Solar Embedding이 가장 높은 검색 정확도를 보였지만, 과금 정책과 사용량 제한을 고려해 최종적으로는 품질과 비용의 균형이 좋은 OpenAI large를 선택했습니다.

LLM 역시 단순 답변 품질보다 의료 정보 서비스에 필요한 근거 기반 응답, 규칙 준수, 위험 상황에서의 상담 안내 여부를 중심으로 비교했습니다.  
그 결과 GPT-4o mini가 비용 효율과 안정적인 답변 품질을 모두 만족한다고 판단하여 최종 적용했습니다.
